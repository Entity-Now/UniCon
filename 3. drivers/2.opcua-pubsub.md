# OPC UA PubSub 驱动 (OPC UA PubSub Driver)

## 概述 (Overview)
本驱动实现了 OPC UA Part 14 规范定义的发布-订阅 (PubSub) 模型通讯。与传统的 Request/Response 模式（如 S7、Modbus）不同，PubSub 提供极低延迟的单向高速数据流传输，特别适合工厂级别的软实时与硬实时通讯。

本驱动深度集成了西门子官方的 **`opc-ua-pubsub-dotnet`** 工业标准库，且已将底层库全面升级适配至 **.NET 10.0** 运行框架，支持双栈接入：
1. **UADP over UDP 多播**: 采用西门子二进制反序列化器，通过 `UadpPubSubDecoder` 动态解包高频多播网络数据，完美映射 MDSP 质量属性与高精度源时间戳。
2. **JSON over MQTT 订阅**: 支持标准的自描述 JSON Payload 及扁平 Key-Value 格式订阅，亦可通过连接参数强制启用 UADP 二进制报文解析。

## 使用方法 (Usage)
本驱动支持双栈接入。通过直接初始化 `OpcUaPubSubDriver` 并配置 URI 连接字符串，驱动会根据协议头自动加载底层的传输组件。

在底层解码器分配上，驱动支持**自适应智能解码**：
- 连接 Scheme 为 `opc.udp://` 时，默认搭载二进制 **`UadpPubSubDecoder`**。
- 连接 Scheme 为 `mqtt://` 时，默认搭载文本 **`JsonPubSubDecoder`**。如果在 MQTT 连接字符串中附加 Query 参数 `?encoding=binary`，驱动会自动切换为二进制解码器，实现 MQTT 下承载 UADP 二进制报文的解析。

## 参数说明 (Parameters)

| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| connectionString | `string` | 指定传输协议与目标地址的 URI 字符串。<br>对于 UDP 组播：`opc.udp://[多播IP]:[端口]`（例如：`opc.udp://224.0.2.14:4840`）<br>对于 MQTT 订阅：`mqtt://[BrokerIP]:[端口]/[Topic路径]`（例如：`mqtt://192.168.1.100:1883/factory/line1`）<br>可以通过附加 `?encoding=binary` 强制在 MQTT 上使用二进制解码。 | 是 | 无 |

## 返回值 (Returns)

通过 `SubscribeAsync` 注册的委托会持续接收到解包后的数据值，返回值为标准工业质量戳结构：

| 类型 | 说明 |
|------|------|
| `DataValue<object>` | 数据值包装器，其包含实际解析出的数值 `Value`、质量戳 `Status` (Good/Bad) 与源时间戳 `SourceTimestamp`。 |

## 使用示例 (Examples)

**示例 1：使用 ConnectionStringBuilder 链式生成 UDP UADP 多播连接**
```csharp
using UniCon.Core.Helpers;
using UniCon.Drivers.OpcUaPubSub;

// 1. 初始化驱动
var pubsubDriver = new OpcUaPubSubDriver(logger);

// 2. 通过 ConnectionStringBuilder 类型安全地生成 UDP 多播连接字符串
string connStr = ConnectionStringBuilder.ForOpcUaPubSub()
    .WithScheme(PubSubScheme.Udp)
    .WithHost("224.0.2.14")
    .WithPort(4840)
    .Build();

// 3. 建立连接并加入 UDP 多播组
await pubsubDriver.ConnectAsync(connStr);

// 4. 订阅感兴趣的变量集 (通过 UadpPubSubDecoder 自动解码)
await pubsubDriver.SubscribeAsync("Temperature", data => {
    Console.WriteLine($"[UDP UADP] 温度: {data.Value}, 质量: {data.Status}, 源时间戳: {data.SourceTimestamp}");
});
```

**示例 2：MQTT 强制开启 UADP 二进制解码订阅**
```csharp
using UniCon.Core.Helpers;
using UniCon.Drivers.OpcUaPubSub;

// 1. 初始化驱动
var mqttPubsubDriver = new OpcUaPubSubDriver(logger);

// 2. 建立 MQTT 订阅，并在 Query 参数中添加 encoding=binary 强制开启西门子二进制解码器
string connStr = ConnectionStringBuilder.ForOpcUaPubSub()
    .WithScheme(PubSubScheme.Mqtt)
    .WithHost("192.168.1.100")
    .WithPort(1883)
    .WithMqttTopic("factory/line1/uadp")
    .WithQueryParam("encoding", "binary")
    .Build();

await mqttPubsubDriver.ConnectAsync(connStr);

// 3. 订阅并接收二进制解包后的数据变动
await mqttPubsubDriver.SubscribeAsync("MotorSpeed", data => {
    Console.WriteLine($"[MQTT UADP] 速度变动: {data.Value}，接收质量: {data.Status}");
});
```

**示例 3：常规 MQTT JSON 格式订阅**
```csharp
using UniCon.Drivers.OpcUaPubSub;

// 1. 初始化驱动
var jsonMqttDriver = new OpcUaPubSubDriver(logger);

// 2. 直接使用普通 MQTT 连接串，默认触发 JsonPubSubDecoder
await jsonMqttDriver.ConnectAsync("mqtt://192.168.0.100:1883/factory/line1/json");

// 3. 订阅接收 JSON 数据
await jsonMqttDriver.SubscribeAsync("Current", data => {
    Console.WriteLine($"[MQTT JSON] 电流: {data.Value}，接收质量: {data.Status}");
});
```
