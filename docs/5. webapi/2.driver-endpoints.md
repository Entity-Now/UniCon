# 动态驱动管理接口 (Dynamic Driver API)

## 概述 (Overview)
`CommunicationService` 提供了系统内通讯驱动的动态生命周期管理（添加、连接、轮询状态、订阅回调及彻底卸载销毁）以及通过统一网关执行的强类型反射式读写交互。通过该服务，外部应用可以通过标准 RESTful 接口进行设备点位的热插拔调试和统一数据集成。

## 使用方法 (Usage)
该组件以单例模式注册在 ASP.NET Core 的依赖注入 (DI) 容器中，并在 WebAPI 程序启动时作为 Minimal APIs 端点的核心处理器使用。外部客户端通过标准的 HTTP 谓词（`GET` / `POST` / `DELETE`）对该接口执行调用。

## 参数说明 (Parameters)

### 1. 驱动注册参数 (CreateDriverRequest)
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| DriverId | string | 驱动的全局唯一标识 ID（不可冲突） | 是 | 无 |
| DriverType | string | 驱动协议类型（支持: `S7`, `Modbus`, `OpcUa`, `Mqtt`, `OpcUaPubSub`） | 是 | 无 |
| ConnectionString | string | 物理连接字符串（详见对应协议驱动文档） | 是 | 无 |

### 2. 通用强类型读取查询参数 (Read Tag Query)
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| address | string | 通信物理地址（例如 S7 的 `DB1.DBD0` 或 Modbus 的 `holding:40001`） | 是 | 无 |
| type | string | 目标值的数据强类型转换指示（如 `int`, `float`, `double`, `bool`, `string`） | 否 | `object` |

### 3. 通用强类型写入参数 (WriteAddressRequest)
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| Address | string | 通信物理地址 | 是 | 无 |
| Value | object | 需要写入的点位数值（JSON 格式数值/布尔/字符串均可） | 是 | 无 |
| TypeHint | string | 强类型映射指示（用于精确转换以防工控协议在底层字节流上解析错位） | 否 | `object` |

---

## 返回值 (Returns)

### 1. 驱动状态对象 (DriverStatus)
| 属性名 | 类型 | 说明 |
|--------|------|------|
| DriverId | string | 驱动唯一 ID |
| DriverType | string | 协议类型 |
| ConnectionString | string | 对应的物理连接字符串 |
| State | string | 运行状态 (`Disconnected`, `Connecting`, `Connected`, `Faulted`, `Disconnecting`) |
| IsConnected | bool | 是否处于正常的物理连通状态 |

### 2. 数据读取返回结构 (DataValue<object>)
| 属性名 | 类型 | 说明 |
|--------|------|------|
| Value | object? | 读取出的物理数据值 |
| Status | string | 数据质量戳状态 (`Good`, `Bad`, `Uncertain`, `Timeout`) |
| SourceTimestamp | DateTime | 数据源生成时的本地/PLC 时间戳 |

## 使用示例 (Examples)

### 示例1：动态添加并连接一个 S7 驱动实例
**HTTP Request:**
```http
POST /api/drivers
Content-Type: application/json

{
  "driverId": "S7_CPU_01",
  "driverType": "S7",
  "connectionString": "CpuType=S71200;Ip=192.168.0.100;Rack=0;Slot=1"
}
```

**HTTP Response (201 Created):**
```json
{
  "driverId": "S7_CPU_01",
  "driverType": "S7",
  "connectionString": "CpuType=S71200;Ip=192.168.0.100;Rack=0;Slot=1",
  "state": "Connected",
  "isConnected": true
}
```

---

### 示例2：通用反射强类型数据读取 (Float 单精度浮点数)
**HTTP Request:**
```http
GET /api/drivers/S7_CPU_01/read?address=DB1.DBD0&type=float
Accept: application/json
```

**HTTP Response (200 OK):**
```json
{
  "value": 45.28,
  "status": 0,
  "sourceTimestamp": "2026-05-17T21:46:12.345Z"
}
```

---

### 示例3：通用反射强类型数据写入 (Boolean 布尔值)
**HTTP Request:**
```http
POST /api/drivers/S7_CPU_01/write
Content-Type: application/json

{
  "address": "DB1.DBX4.0",
  "value": true,
  "typeHint": "bool"
}
```

**HTTP Response (200 OK):**
```json
{
  "success": true,
  "data": {
    "value": true,
    "status": 0,
    "sourceTimestamp": "2026-05-17T21:47:05.123Z"
  },
  "headers": {},
  "statusCode": 0,
  "errorMessage": null
}
```

---

### 示例4：订阅点位与轮询最新值缓存
**HTTP Request - 创建订阅:**
```http
POST /api/drivers/S7_CPU_01/subscribe
Content-Type: application/json

{
  "address": "DB1.DBD8"
}
```

**HTTP Response (200 OK):**
```json
{
  "message": "Successfully subscribed to 'DB1.DBD8' on driver 'S7_CPU_01'."
}
```

**HTTP Request - 获取缓存最新数据:**
```http
GET /api/drivers/S7_CPU_01/subscriptions
Accept: application/json
```

**HTTP Response (200 OK):**
```json
[
  {
    "driverId": "S7_CPU_01",
    "address": "DB1.DBD8",
    "value": 102.5,
    "quality": "Good",
    "timestamp": "2026-05-17T21:49:15.823Z"
  }
]
```

---

### 示例5：注销、断开并销毁驱动实例
**HTTP Request:**
```http
DELETE /api/drivers/S7_CPU_01
```

**HTTP Response (200 OK):**
```json
{
  "message": "Driver 'S7_CPU_01' has been disconnected, unregistered, and disposed."
}
```
