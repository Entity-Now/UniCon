# 统一集中式主动轮询订阅与缓存引擎 (Centralized Polling & Caching Engine)

## 概述 (Overview)
对于 S7、Modbus 等南向物理总线并不支持发布订阅机制的工控协议，统一主动轮询订阅引擎在 `UniCon.Core` 基础层实现了一套精巧的主动适配方案。

在高并发架构中，该引擎采用 **集中化轮询调度器 (Central Polling Coordinator)** 调度设计：
1. **单一主后台线程模型**：弃用“每个地址创建一个后台 Task.Run 循环”的原有低效机制，每个物理驱动仅运行一个统一的共享高精度调度任务。
2. **去重与批量读取优化**：自动对期满 (Due) 的订阅项按物理寄存器地址进行去重聚合，并执行 `ReadBatchAsync` 批量分组网络查询，最小化 PLC 物理网络延迟与 CPU 线程调度上下文切换开销。
3. **结构化监控管理**：允许通过结构化的 `UniconSubscription` 类进行细粒度控制，支持按 ID 取消订阅、查询监控列表与多订阅共享同一寄存器点位。

---

## 使用方法 (Usage)
引擎自动继承并内置在所有继承自 `DriverBase` 的驱动实例中：

**方法 1：极简地址订阅 (保持 100% 向下兼容)**
```csharp
// 1. 发起极简主动变化订阅 (使用 1000ms 默认频率与 ExceptionBased 模式)
await driver.SubscribeAsync("DB1.DBD0", (dataValue) =>
{
    Console.WriteLine($"收到变更点位: {dataValue.Value}, 质量: {dataValue.Status}");
});

// 2. 取消该地址下的所有订阅
await driver.UnsubscribeAsync("DB1.DBD0");
```

**方法 2：结构化精准订阅 (高定制，推荐)**
```csharp
// 1. 构建结构化订阅选项 (支持独立配置扫描周期与扫描模式)
var subscription = new UniconSubscription
{
    Address = "DB1.DBD0",
    ScanRateMs = 500,
    ScanMode = UniconScanMode.ExceptionBased,
    Callback = (dataValue) =>
    {
        Console.WriteLine($"点位变更通知: {dataValue.Value}");
    }
};

// 2. 注册并获取唯一的订阅 ID
string subId = await driver.SubscribeAsync(subscription);

// 3. 获取驱动中当前的全部活动订阅列表
IEnumerable<UniconSubscription> activeSubs = driver.GetSubscriptions();

// 4. 根据唯一的订阅 ID 平滑注销特定回调，而不影响该地址上的其他订阅项
await driver.UnsubscribeByIdAsync(subId);
```

## 参数说明 (Parameters)

### `UniconSubscription` 属性说明

| 属性名 | 类型 | 说明 | 是否必选 | 默认值 |
|--------|------|------|----------|--------|
| Id | `string` | 订阅唯一标识符，系统在实例化时自动生成 `Guid` | 否 | 随机 Guid |
| Address | `string` | 需要轮询订阅的物理设备寄存器地址路径 (如 `"DB1.DBD0"`, `"holding:1"`) | 是 | 无 |
| Callback | `Action<DataValue<object>>` | 订阅点位发生变更或触发刷新时的北向通知回调委托 | 是 | 无 |
| ScanRateMs | `int` | 该订阅项被调度扫描的时间间隔周期 (单位: 毫秒) | 否 | `1000` |
| ScanMode | `UniconScanMode` | 推送模式：`ExceptionBased` (变更推送) 或 `Polled` (周期刷屏) | 否 | `ExceptionBased` |

---

## 返回值 (Returns)

### `SubscribeAsync(UniconSubscription)` 返回值
| 类型 | 说明 |
|------|------|
| `Task<string>` | 异步注册任务，挂载成功后返回该订阅的唯一 ID 令牌 (Token)，用于精准注销 |

## 使用示例 (Examples)

**示例 1：对 S7 物理点位执行变化订阅 (ExceptionBased 异常过滤模式)**
```csharp
// 获取 S7 驱动实例
var s7Driver = driverRegistry.Get("S7_PLC_01");

// 订阅反应釜温度，设置 500ms 周期扫描，使用 ExceptionBased 模式（仅在温度改变时才触发打印）
await s7Driver.SubscribeAsync("DB1.DBD0", (dataValue) =>
{
    Console.WriteLine($"【温度变更】实时值: {dataValue.Value} °C, 时间: {dataValue.SourceTimestamp:HH:mm:ss.fff}");
}, 500, UniconScanMode.ExceptionBased);
```

**示例 2：实现自定义 Redis 兼容缓存提供者以替换内置 Memory 缓存**
```csharp
using System.Threading;
using System.Threading.Tasks;
using UniCon.Core.Caching;
using UniCon.Core.Models;
using StackExchange.Redis;

namespace UniCon.Infrastructure.Caching
{
    /// <summary>
    /// 自定义分布式 Redis 物理缓存实现
    /// </summary>
    public class MyCustomRedisCacheProvider : IUniconCacheProvider
    {
        private readonly IDatabase _db;

        public MyCustomRedisCacheProvider(IConnectionMultiplexer redis)
        {
            _db = redis.GetDatabase();
        }

        public async Task<DataValue<object>?> GetAsync(string driverId, string address, CancellationToken ct = default)
        {
            string key = $"unicon:cache:{driverId}:{address}";
            string? json = await _db.StringGetAsync(key);
            return json != null ? System.Text.Json.JsonSerializer.Deserialize<DataValue<object>>(json) : null;
        }

        public async Task SetAsync(string driverId, string address, DataValue<object> dataValue, CancellationToken ct = default)
        {
            string key = $"unicon:cache:{driverId}:{address}";
            string json = System.Text.Json.JsonSerializer.Serialize(dataValue);
            await _db.StringSetAsync(key, json);
        }

        public async Task RemoveAsync(string driverId, string address, CancellationToken ct = default)
        {
            string key = $"unicon:cache:{driverId}:{address}";
            await _db.KeyDeleteAsync(key);
        }
    }
}
```
