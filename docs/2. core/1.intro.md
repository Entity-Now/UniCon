# 核心架构 (Core Architecture)

## 概述
Core 模块是 UniCon 的心脏，定义了系统的全部契约、模型结构、基础自愈服务、统一缓存与 IOC 注册入口。

## 关键功能
- **接口定义**: `IUniconDriver` 定义了统一的通讯契约，包括即时读写、批量读写、探活及订阅。
- **消息模型**: `UniconRequest` 与 `UniconResponse` 实现了高阶的结构化交互，内部数据由 `DataValue<T>` 包装，附带高精度源时间戳与质量戳。
- **驱动注册中心 (`IDriverRegistry`)**: 实现驱动实例在全系统内的统一共享、动态查询与状态轮询。
- **连接自愈管理器 (`IConnectionManager`)**: 全生命周期托管所有被添加驱动实例的 Socket/通道连接，并与驱动内部的 Watchdog 指数退避重连机制联动实现自主自愈。
- **对象-设备映射引擎 (`OdmEngine`)**: 类似于工业领域的 ORM，支持将业务实体属性通过强类型特性特性映射到设备通道地址，提供免硬编码的动态实体自动填充读写服务。
- **统一主动轮询订阅与自愈缓存引擎**: 在核心层支持主动对 S7、Modbus 等无主动上报机制协议进行后台 Task 定时扫描与值变更检测（UniconScanMode），通过可插拔的 `IUniconCacheProvider` 进行缓存写入与事件回调。
- **开箱即用集成 (`ServiceCollectionExtensions`)**: 针对类库进行 NuGet 友好度封装，提供一键式 `AddUniCon()` 依赖注入扩展，以单例模式将注册中心、连接管理器、缓存管理器等接入 IOC 容器中。
- **连接构建器 (`ConnectionStringBuilder`)**: 提供强类型、流式 API 来动态组装各工控驱动的连接字符串，规范配置解析。
