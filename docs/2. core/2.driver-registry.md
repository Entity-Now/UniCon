# 驱动注册与创建中心 (Driver Registry & Factory)

## 概述 (Overview)
`IDriverRegistry`（驱动注册与创建中心）是 UniCon 系统的核心基础设施，它合并了**实例注册中心 (Instance Registry)** 与 **解耦装配工厂 (Dynamic Factory)** 的双重职责，彻底消除了多组件设计冗余：
1. **活跃实例生命周期管理 (Registry)**：负责维护 UniCon 框架中所有当前运行、活跃的物理通讯驱动实例（如 `S7Driver`、`ModbusDriver` 等），支持在 WebAPI 交互层、定时调度系统 (Job System) 和映射引擎 (OdmEngine) 中实现驱动实例的全局共享与统一注销。
2. **免配置反射自动装配 (Factory & Reflection)**：
   - 提供了全新的 `[UniconDriver]` 特性标记与程序集主动扫描引擎 `DiscoverAndRegisterDrivers()`。
   - 启动时自动搜索应用程序域及物理运行目录下的所有 `UniCon.Drivers.*.dll` 驱动模块，免去了人工编写 `RegisterDriverType` 的硬编码行为，实现了真“即插即用 (Plug-and-Play)”的南向生态。
   - 利用 .NET 核心的 `ActivatorUtilities` 自动装配机制，对开发者和使用者完全隐蔽了南向驱动庞杂的环境依赖参数（如 `IUniconCacheProvider`、`INetworkMonitor`）。

## 使用方法 (Usage)

### 1. 标记驱动类
在具体南向驱动类（如 `S7Driver`）上贴上 `[UniconDriver]` 特性，并指明其别名简称：
```csharp
using UniCon.Core;

namespace UniCon.Drivers.S7
{
    [UniconDriver("S7")]
    public class S7Driver : DriverBase
    {
        // 驱动逻辑实现...
    }
}
```

### 2. 服务注册与启动自扫描
在 .NET 依赖注入容器中注册并触发自动发现：
```csharp
// 1. 注入核心服务
builder.Services.AddUniCon(); // 内部已包含 services.AddSingleton<IDriverRegistry, DriverRegistry>();

// 2. 在 app 构建完成后，调用反射自发现机制完成装配
var app = builder.Build();
var driverRegistry = app.Services.GetRequiredService<IDriverRegistry>();
driverRegistry.DiscoverAndRegisterDrivers(); // 一键自发现，从此 Program.cs 中没有任何具体驱动的 compile-time 依赖！
```

## 参数与接口说明 (API Reference)

### 1. 运行中驱动实例管理 (Registry API)

#### `Register`
将一个外部构建好的驱动实例手动托管至注册中心。
- **参数 `driver`** (`IUniconDriver`): 需要注册的驱动实例。

#### `Unregister`
从注册中心中注销该驱动实例，停止其生命周期托管。
- **参数 `driverId`** (`string`): 目标驱动 ID。
- **返回值** (`bool`): 注销成功返回 `true`，若不存在该驱动则返回 `false`。

#### `Get`
检索并获取当前已托管的驱动活跃实例。
- **参数 `driverId`** (`string`): 目标驱动 ID。
- **返回值** (`IUniconDriver?`): 活跃实例引用，未找到返回 `null`。

#### `GetAll`
获取当前中心正在托管运行的全部驱动实例集合。
- **返回值** (`IEnumerable<IUniconDriver>`): 全量只读集合。

---

### 2. 驱动工厂别名配置与实例化 (Factory API)

#### `DiscoverAndRegisterDrivers`
反射发现核心：扫描当前 AppDomain 中已加载的程序集，并主动加载运行目录（BaseDirectory）下所有以 `UniCon.Drivers.*.dll` 命名的物理程序集文件，并提取类上贴有 `[UniconDriver]` 的有效驱动，完成工厂别名注册。

#### `RegisterDriverType`
手动注册驱动简称别名到对应的真实驱动实现类型（通常在特定反射无法触及的单元测试或 Mock 场景中使用）。
- **参数 `driverType`** (`string`): 驱动简称（如 `"S7"`）。
- **参数 `implementationType`** (`Type`): 具体驱动运行时类型。

#### `CreateDriver`
根据注册的简称别名和自定义 ID，自动从依赖注入容器拉取必要参数（如 `IUniconCacheProvider`, `INetworkMonitor`），并优先使用指定局部参数完成装配。
- **参数 `driverType`** (`string`): 注册的驱动简称别名（如 `"S7"`）。
- **参数 `driverId`** (`string`): 此驱动实例的唯一标识 ID。
- **返回值** (`IUniconDriver`): 成功装配好的驱动实例。

#### `CreateDriver<T>`
以泛型强类型方式一键装配、创建指定驱动，**无需提前进行别名扫描或注册**。
- **类型参数 `T`** (实现 `IUniconDriver` 的具体类)。
- **参数 `driverId`** (`string`): 此驱动实例的唯一标识 ID。
- **返回值** (`T`): 成功装配好的强类型驱动实例。

## 使用示例 (Examples)

**示例 1：泛型零摩擦创建并注册托管驱动**
```csharp
public class PlcManager
{
    private readonly IDriverRegistry _driverRegistry;
    private readonly IConnectionManager _connectionManager;

    public PlcManager(IDriverRegistry driverRegistry, IConnectionManager connectionManager)
    {
        _driverRegistry = driverRegistry;
        _connectionManager = connectionManager;
    }

    public async Task InitializeLine1PlcAsync()
    {
        // 1. 泛型一键创建并自动装配 Log/Cache/NetworkMonitor 依赖
        var plcDriver = _driverRegistry.CreateDriver<S7Driver>("PLC_01");

        // 2. 存入注册中心托管
        _driverRegistry.Register(plcDriver);

        // 3. 激活连接
        await _connectionManager.RegisterDriverAsync(plcDriver, "ip=192.168.1.100;cputype=S71200");
    }
}
```

**示例 2：基于配置别名动态拉起驱动 (真·即插即用模式)**
```csharp
// 运行时从 DB 或配置文件读出 "S7" 和 "PLC_Line2"
string configType = "S7";
string configId = "PLC_Line2";

// 启动时已调用 DiscoverAndRegisterDrivers()，此处直接动态生成
IUniconDriver driver = _driverRegistry.CreateDriver(configType, configId);
_driverRegistry.Registry.Register(driver);
```
