# Siemens S7 驱动 (Siemens S7 Driver)

## 概述 (Overview)
西门子 S7 驱动实现了基于以太网对西门子全系列 PLC（如 S7-200 Smart, S7-300, S7-1200, S7-1500 等）的直连通讯。底层基于高效的 `S7.Net` 开源库构建，利用 UniCon 的自愈与并发机制，保证了工业现场数据采集的极高稳定性。

## 使用方法 (Usage)
通过标准 DI 容器注入或实例化 `S7Driver`，使用以分号分隔的 KV 连接字符串建立连接，并执行读写操作。

## 参数说明 (Parameters)

### 连接字符串参数 (ConnectionString)
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| CpuType | `string` | 支持的值包括：`S7200`, `S7300`, `S7400`, `S71200`, `S71500` | 是 | 无 |
| Ip | `string` | PLC 的 IP 地址 | 是 | 无 |
| Rack | `short` | PLC 的机架号 (Rack) | 否 | 0 |
| Slot | `short` | PLC 的插槽号 (Slot) | 否 | 1 |
| Timeout / ReadTimeout | `int` | 读取网络超时时间 (毫秒) | 否 | 0 (使用系统默认值) |
| WriteTimeout | `int` | 写入网络超时时间 (毫秒) | 否 | 0 (使用系统默认值) |

### 读写方法参数 (ReadAsync/WriteAsync)
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| Address | `string` | S7 寄存器绝对地址，如 `DB1.DBD0`（数据块 1，偏移量 0 的双字），`M0.0`，`I0.0` | 是 | 无 |
| value | `T` | 写入的具体值（仅 `WriteAsync` 需要） | 是 | 无 |

## 返回值 (Returns)

### ReadAsync 返回值
| 类型 | 说明 |
|------|------|
| `UniconResponse<T>` | 响应包装器，内含读取到的变量质量与实际值，失败时附带异常状态码与具体错误消息 |

### WriteAsync 返回值
| 类型 | 说明 |
|------|------|
| `UniconResponse<bool>` | 响应包装器，表示写入动作是否成功。若为 `true` 代表成功写入，若失败则附带错误状态码 |

## 使用示例 (Examples)

**示例 1：读取西门子 S7-1200 实数**
```csharp
using UniCon.Core.Models;
using UniCon.Drivers.S7;

var s7Driver = new S7Driver("Line1_PLC", logger);
await s7Driver.ConnectAsync("CpuType=S71200;Ip=192.168.0.10;Rack=0;Slot=1");

var req = new UniconRequest { Address = "DB1.DBD0" }; // DB1.DBD0 为一个 Float 值
var res = await s7Driver.ReadAsync<float>(req);

if (res.Success && res.Data != null)
{
    Console.WriteLine($"当前读取值: {res.Data.Value}, 质量状态: {res.Data.Status}");
}
```
