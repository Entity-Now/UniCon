# Modbus 驱动 (Modbus Driver)

## 概述 (Overview)
Modbus 驱动实现了标准的 Modbus TCP 通讯协议。能够访问 PLC、变频器、智能仪表等广泛设备的线圈 (Coil)、离散输入 (Discrete Input)、保持寄存器 (Holding Register) 及输入寄存器 (Input Register)。底层基于开源 `EasyModbusTCP` 实现，并经由 UniCon 基类封装了串行队列并发控制。

## 使用方法 (Usage)
通过 `ModbusDriver` 实例并传入 IP 和端口连接，并使用格式为 `[寄存器类型]:[地址]` 的地址串读取或写入。

## 参数说明 (Parameters)

### 连接字符串参数 (ConnectionString)
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| ip | `string` | Modbus 从站 (Slave) 或服务器的 IP 地址 | 是 | 无 |
| port | `int` | Modbus 服务的 TCP 端口（标准通常为 502） | 否 | 502 |
| unitid / slaveid | `byte` | 从站单元标识符 (Slave ID / Unit ID) | 否 | 1 |
| timeout / connectiontimeout | `int` | 通讯及连接超时时间 (毫秒) | 否 | 2000 |

### 读写方法参数 (ReadAsync/WriteAsync)
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| Address | `string` | 格式为 `[类型]:[地址]`。<br>支持类型有：<br> - `coil` (线圈)<br> - `discrete` (离散输入)<br> - `holding` (保持寄存器)<br> - `input` (输入寄存器) | 是 | 无 |
| value | `T` | 写入具体值（只适用于 `holding` 与 `coil` 类型） | 是 | 无 |

## 返回值 (Returns)

### ReadAsync 返回值
| 类型 | 说明 |
|------|------|
| `UniconResponse<T>` | 响应包装器，内含读取的值及其质量状态戳，若网络超时或解析错误则带有失败说明 |

### WriteAsync 返回值
| 类型 | 说明 |
|------|------|
| `UniconResponse<bool>` | 响应包装器，返回是否成功写入寄存器 |

## 使用示例 (Examples)

**示例 1：读取保持寄存器 40001 (地址偏移 1)**
```csharp
using UniCon.Core.Models;
using UniCon.Drivers.Modbus;

var modbusDriver = new ModbusDriver("Modbus_Meter", logger);
await modbusDriver.ConnectAsync("ip=192.168.0.50;port=502");

// 读取保持寄存器地址 1
var req = new UniconRequest { Address = "holding:1" };
var res = await modbusDriver.ReadAsync<int>(req);

if (res.Success && res.Data != null)
{
    Console.WriteLine($"当前寄存器值: {res.Data.Value}");
}
```
