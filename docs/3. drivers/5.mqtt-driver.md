# MQTT 消息驱动 (MQTT Message Driver)

## 概述 (Overview)
MQTT 消息驱动主要用于轻量级工业 IoT 消息传输及事件驱动的发布/订阅模式。驱动底层依赖 `MQTTnet` 库实现，能够在多任务和并发环境中安全收发工业消息，特别适用于网关上云和分布式数据采集。

## 使用方法 (Usage)
通过 `MqttDriver` 进行物理连接，利用 `SubscribeAsync` 绑定订阅主题与数据回调，或利用 `WriteAsync` 往特定主题发送消息 payload。

## 参数说明 (Parameters)

### 连接字符串参数 (ConnectionString)
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| server / host | `string` | MQTT Broker 服务的 IP 或主机名 | 是 | 无 |
| port | `int` | MQTT Broker 端口号 | 否 | 1883 |
| clientid | `string` | 客户端唯一连接 ID | 否 | 随机生成 |
| username / user | `string` | 账户授权登录用户名 | 否 | 空 (匿名登录) |
| password / pwd | `string` | 账户授权登录密码 | 否 | 空 |
| cleansession | `bool` | 是否清除会话 (Clean Session) | 否 | `true` |
| keepalive | `int` | 心跳维持间隔时间 (秒) | 否 | 15 |
| usetls / tls | `bool` | 是否启用 SSL/TLS 安全连接 | 否 | `false` |

### 方法参数 (SubscribeAsync/WriteAsync)
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| address | `string` | MQTT 主题名称 (Topic)，支持使用通配符如 `+` 和 `#` | 是 | 无 |
| callback | `Action<DataValue<object>>` | 接收到消息变动时的通知委托（仅订阅需要） | 是 | 无 |
| value | `T` | 要发布的具体消息载荷字符串（仅写入时需要） | 是 | 无 |

## 返回值 (Returns)

### ReadAsync 返回值
| 类型 | 说明 |
|------|------|
| `UniconResponse<T>` | **不支持直接读取**。固定返回错误状态码 `405` (Method Not Allowed) |

### WriteAsync 返回值
| 类型 | 说明 |
|------|------|
| `UniconResponse<bool>` | 响应包装器，若发布成功则为 `true`，失败则包含异常状态码 |

## 使用示例 (Examples)

**示例 1：订阅特定设备遥测主题**
```csharp
using UniCon.Core.Models;
using UniCon.Drivers.Mqtt;

var mqttDriver = new MqttDriver("Cloud_Broker", logger);
await mqttDriver.ConnectAsync("server=192.168.1.200;clientid=unicon_gateway_01");

// 订阅指定主题并挂载事件处理
await mqttDriver.SubscribeAsync("devices/meter01/telemetry", data => {
    Console.WriteLine($"收到仪表数据: {data.Value}，时间戳: {data.SourceTimestamp}");
});
```
