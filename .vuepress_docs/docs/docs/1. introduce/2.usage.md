# 快速入门与集成指南

本指南将介绍如何在第三方项目中引用 UniCon 框架并快速实现工业设备通讯与采集调度。

## 1. 安装与引用

UniCon 采用高度解耦的**插件化与模块化**架构设计。核心框架与各个具体物理通信驱动均已在 NuGet 仓库中作为独立的包发布。

::: tip 💡 我需要依次安装所有的 NuGet 包吗？
**不需要。**
您只需且必须安装核心依赖包 **`UniCon.Core`**，然后根据您实际的工业场景，**仅选择性地安装需要的特定协议驱动包**即可。例如，若您的项目仅需要采集 S7 PLC 数据和 Modbus 寄存器，则只需安装 `UniCon.Core`、`UniCon.Drivers.S7` 和 `UniCon.Drivers.Modbus`。这样可以避免引入多余的外部三方依赖（如 OPC UA 或 MQTTnet 库），保持您的业务项目极简、纯净与高效。
:::

### 📦 核心调度引擎 (必选)
* **.NET CLI**: 
  ```bash
  dotnet add package UniCon.Core
  ```
* **Package Manager 控制台**: 
  ```powershell
  Install-Package UniCon.Core
  ```

### 🔌 物理通讯驱动包 (按需选择)

| 驱动名 | 说明 | .NET CLI 安装命令 | Package Manager 安装命令 |
| :--- | :--- | :--- | :--- |
| **`UniCon.Drivers.S7`** | 西门子全系列 PLC (S7-1200/1500/300) | `dotnet add package UniCon.Drivers.S7` | `Install-Package UniCon.Drivers.S7` |
| **`UniCon.Drivers.OpcUa`** | 高性能 OPC UA 客户端通讯 | `dotnet add package UniCon.Drivers.OpcUa` | `Install-Package UniCon.Drivers.OpcUa` |
| **`UniCon.Drivers.Modbus`** | 标准 Modbus TCP / RTU 物理设备 | `dotnet add package UniCon.Drivers.Modbus` | `Install-Package UniCon.Drivers.Modbus` |
| **`UniCon.Drivers.Mqtt`** | 轻量级 MQTT 消息队列发布与订阅 | `dotnet add package UniCon.Drivers.Mqtt` | `Install-Package UniCon.Drivers.Mqtt` |
| **`UniCon.OpcUaPubSub.Binary`** | OPC UA 组播二进制流自定义解析器 | `dotnet add package UniCon.OpcUaPubSub.Binary` | `Install-Package UniCon.OpcUaPubSub.Binary` |
| **`UniCon.OpcUaPubSub.Client`** | OPC UA 组播订阅 UDP 通讯客户端 | `dotnet add package UniCon.OpcUaPubSub.Client` | `Install-Package UniCon.OpcUaPubSub.Client` |

## 2. 基础通讯示例

以下展示如何初始化一个 S7 驱动并读取数据。

```csharp
using UniCon.Core;
using UniCon.Core.Caching;
using UniCon.Core.Models;
using UniCon.Drivers.S7;
using Microsoft.Extensions.Logging;

// 1. 初始化依赖 (v2 要求 DI 注入 CacheProvider)
var logger = LoggerFactory.Create(b => b.AddConsole()).CreateLogger("S7");
IUniconCacheProvider cache = new MemoryCacheProvider();

// 2. 初始化驱动
var s7Driver = new S7Driver("PLC_01", logger, cache);

// 3. 开启自愈重连配置 (默认开启)
s7Driver.EnableAutoReconnect = true;
s7Driver.InitialRetryIntervalMs = 2000;

// 4. 建立物理连接
await s7Driver.ConnectAsync("CpuType=S71200;Ip=192.168.0.10;Rack=0;Slot=1");

// 5. 结构化读取请求
var request = new UniconRequest { Address = "DB1.DBD0" };
var response = await s7Driver.ReadAsync<float>(request);

if (response.Success)
{
    Console.WriteLine($"Value: {response.Data.Value}");
    Console.WriteLine($"Quality: {response.Data.Quality}");
}
```

## 3. 集成到 ASP.NET Core 与任务调度

UniCon 提供了开箱即用的扩展方法，一键将注册中心、连接管理器与缓存注入容器。

### 注册服务 (`Program.cs`)
```csharp
// 添加基础系统 (包含 IDriverRegistry, IConnectionManager, IUniconCacheProvider 等)
builder.Services.AddUniCon();

// 若需集成定时任务 (Job 系统)
builder.Services.AddQuartz();
builder.Services.AddQuartzHostedService();
builder.Services.AddSingleton<JobScheduler>();
```

### 使用主动采集调度引擎 (v2)
v2 引擎提供极高的并发与无 LINQ 损耗的运行时调度。

```csharp
// 在后台 Service 中订阅
await s7Driver.SubscribeAsync(new UniconSubscription
{
    Address = "DB1.DBD4",
    ScanRateMs = 500, // 每 500ms 调度一次网络请求
    ScanMode = UniconScanMode.ExceptionBased, // 仅值或质量变化时触发回调
    Callback = async dataValue => 
    {
        Console.WriteLine($"[{dataValue.ServerTimestamp}] 数据更新: {dataValue.Value}");
        await Task.CompletedTask;
    }
});
```

## 4. 批量操作与数据查询

### 批量读取（底层自动合并连续点位）
```csharp
var requests = new List<UniconRequest>
{
    new() { Address = "DB1.DBD0" },
    new() { Address = "DB1.DBD4" }
};

var results = await s7Driver.ReadBatchAsync(requests);
foreach (var res in results)
{
    if (res.Success) Console.WriteLine(res.Data.Value);
}
```

### 获取运行统计
```csharp
var stats = s7Driver.GetStatistics(500, UniconScanMode.ExceptionBased);
Console.WriteLine($"该调度组丢包率: {stats?.ErrorRate:P2}");
```
