# 通讯联动任务 (CommunicationJob)

## 使用方法 (Usage)
用于定时执行 PLC 或其他工业设备的读写操作。

## 参数签名 (Parameters)
通过 `JobDataMap` 传递：
- `Job_Comm_DriverId` (string): 已注册的驱动 ID。
- `Job_Comm_Address` (string): 通讯地址。
- `Job_Comm_Operation` (string): "Read" 或 "Write"。
- `Job_Comm_Value` (object): 写入时的值。
- `Job_Comm_DataType` (string): 数据类型全名 (例如 "System.Single")。

## 返回签名 (Returns)
- **读取结果**: 记录在日志中 `Read Response: {UniconResponse}`。
- **写入结果**: 记录在日志中 `Write Response: {UniconResponse}`。

## 使用案例 (Examples)
```csharp
// 案例 1：定时读取 PLC 温度
await jobScheduler.ScheduleJobAsync<CommunicationJob>(
    jobId: "ReadTemperature",
    cronExpression: "0/10 * * * * ?", // 每 10 秒
    data: new JobDataMap 
    { 
        [JobDataKeys.CommDriverId] = "PLC_01",
        [JobDataKeys.CommAddress] = "DB1.DBD0",
        [JobDataKeys.CommOperation] = "Read",
        [JobDataKeys.CommDataType] = "System.Single"
    }
);

// 案例 2：定时重置计数器 (写入操作)
await jobScheduler.ScheduleJobAsync<CommunicationJob>(
    jobId: "ResetCounter",
    cronExpression: "0 0 0 * * ?", // 每天零点
    data: new JobDataMap 
    { 
        [JobDataKeys.CommDriverId] = "PLC_01",
        [JobDataKeys.CommAddress] = "DB1.DBW10",
        [JobDataKeys.CommOperation] = "Write",
        [JobDataKeys.CommValue] = 0,
        [JobDataKeys.CommDataType] = "System.Int16"
    }
);
```
