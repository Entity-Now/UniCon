# HTTP 任务 (HttpJob)

## 使用方法 (Usage)
通过 `JobScheduler` 安排 HTTP 任务。支持 GET/POST 等方法，可配置 Header、Query 和 Body。

## 参数签名 (Parameters)
通过 `JobDataMap` 传递以下键值：
- `Job_Http_Url` (string): 目标 URL。
- `Job_Http_Method` (string): 请求方法 (默认 GET)。
- `Job_Http_Headers` (string): JSON 格式的字典 (Dictionary<string, string>)。
- `Job_Http_QueryParams` (string): JSON 格式的字典 (Dictionary<string, string>)。
- `Job_Http_Body` (string): 请求体内容。

## 返回签名 (Returns)
- 该任务为异步执行，结果记录在日志系统中：
    - 成功：记录 `HttpJob response: {StatusCode}`。
    - 失败：记录异常详情。

## 使用案例 (Examples)
```csharp
// 安排一个每 5 分钟向 WebHook 推送数据的任务
await jobScheduler.ScheduleJobAsync<HttpJob>(
    jobId: "PushDataToCloud", 
    cronExpression: "0 0/5 * * * ?", 
    data: new JobDataMap 
    { 
        [JobDataKeys.HttpUrl] = "https://api.example.com/v1/data", 
        [JobDataKeys.HttpMethod] = "POST",
        [JobDataKeys.HttpBody] = "{\"deviceId\": \"PLC_01\", \"status\": \"running\"}",
        [JobDataKeys.HttpHeaders] = "{\"Authorization\": \"Bearer token123\"}"
    }
);
```
