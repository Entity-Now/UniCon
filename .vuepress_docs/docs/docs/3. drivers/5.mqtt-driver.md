# MQTT 消息驱动 (MQTT Message Driver)

## 概述 (Overview)
MQTT 消息驱动主要用于轻量级工业 IoT 消息传输及事件驱动的发布/订阅模式。驱动底层依赖 `MQTTnet` 库实现，能够在并发环境中安全收发工业消息。

**注意**：MQTT 属于 Push（推送）型协议，其订阅直接映射为底层 Broker 的 Topic 订阅，**不经过 `ScanScheduler` 主动轮询调度**，而是由网络事件触发。

## 使用方法 (Usage)
通过 `MqttDriver` 进行物理连接，利用 `SubscribeAsync` 绑定订阅主题，或利用 `WriteAsync` 往特定主题发送消息 payload。

## 参数说明 (Parameters)

### 连接字符串参数
| 参数名 | 类型 | 说明 | 是否必填 | 默认值 |
|--------|------|------|----------|--------|
| server / host | `string` | MQTT Broker 服务的 IP 或主机名 | 是 | 无 |
| port | `int` | MQTT Broker 端口号 | 否 | 1883 |
| clientid | `string` | 客户端唯一连接 ID | 否 | 随机生成 |
| username / user | `string` | 账户授权登录用户名 | 否 | 空 |
| password / pwd | `string` | 账户授权登录密码 | 否 | 空 |
| cleansession | `bool` | 是否清除会话 | 否 | `true` |
| keepalive | `int` | 心跳维持间隔时间 (秒) | 否 | 15 |
| usetls / tls | `bool` | 是否启用 SSL/TLS 安全连接 | 否 | `false` |

## 返回值 (Returns)

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `ReadAsync` | `UniconResponse<T>` | **固定报错 405**，MQTT 不支持主动单次拉取读取 |
| `WriteAsync` | `UniconResponse<bool>` | 成功发布消息至 Broker 则返回 true |
| `SubscribeAsync` | `Task<string>` | 返回 Topic 本身作为订阅 ID |

## 使用示例 (Examples)

**示例 1：订阅特定设备遥测主题**
```csharp
var mqttDriver = new MqttDriver("Cloud_Broker", logger, cacheProvider);
await mqttDriver.ConnectAsync("server=192.168.1.200;clientid=gateway_01");

// 直接传入 Func 作为回调（支持通配符）
await mqttDriver.SubscribeAsync("devices/+/telemetry", async dataValue => 
{
    Console.WriteLine($"收到消息: {dataValue.Value}");
    Console.WriteLine($"到达时间: {dataValue.ServerTimestamp}");
    await Task.CompletedTask;
});
```

**示例 2：向云端发布报文**
```csharp
var req = new UniconRequest { Address = "gateway/status" };
await mqttDriver.WriteAsync(req, "{\"status\": \"online\"}");
```
